# Os arquivos Markdown, devem sempre seguir um padrão, a não ser que seja orientado do contrário.

Os arquivos devem ter em seu ínio pelo menos o grupo de badge apresentados a baixo ajustado conforme o conteúdo e conforme o repositório do git, obtenha o repositório para fazer os ajustes necessários.

Os arquivos markdown deverão ter um cabeçalho com gif animado, com o título sobre ele.

A licença deve ser ajustada conforme orientação.

Badges desnecessários devem ser removidos. Por exemplo se o arquivo não tem video não precisa do badges "Video Content", se o documento não é um tutorial, guia ou manual não precisa do badge. Estando "script", "_script", e similares o tipo deve ser "Scripts". Se estiver em uma pasta que é uma coleção de ebooks, datasheets, guias e outros similares deve ser "Biblioteca" e assim por diante.

```text
![visitors](https://visitor-badge.laobi.icu/badge?page_id=<organization>.<repositorio>.<nome_arquivo>)
[![License: CC BY-SA 4.0](https://img.shields.io/badge/License-CC_BY--SA_4.0-blue.svg)](https://creativecommons.org/licenses/by-sa/4.0/)
![Language: Portuguese](https://img.shields.io/badge/Language-Portuguese-brightgreen.svg)
![Video Content](https://img.shields.io/badge/Content-Video%20Lessons-red)
![Educational](https://img.shields.io/badge/Type-Educational-green)
![Status](https://img.shields.io/badge/Status-Ongoing-yellow)
```

Os títulos devem receber emoticons na primeira coluna para destaca-los e orientar do que se tratam.

Quando encontrar uma URL crie o código para gerar o link no local que ele está, se for num paragrafo use seu título, se for souto no texto gere uma lista não numerda, e coloque seu título, consulte na internet para obter o tíulo.

No rodapé do arquivo, deve ter um resumo de uma linha do objetivo do arquivo, qual Repositório ele pertence e qual organização. Abaixo deve ter a data de criação do arquivo e o código da licença escolhida.

Os arquivos markdown devem ter no rodapé um git animado para finalizar o conteúdo da página.
